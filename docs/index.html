Home/Layer

Layer Component

Assigns an entity to one of 32 collision/rendering layers and controls which other layers it interacts with. The layer system drives both PhysX collision filtering and Physics.raycast queries.

A LayerComponent is created automatically when you first call Layer.setLayer or Layer.setMask. Entities without one are treated as layer 0 with mask 0xFFFFFFFF (interacts with everything).
FunctionReturnsDescription
Layer.hasLayer(e)boolReturns true if the entity has a LayerComponent
Layer.add(e)Add a LayerComponent with defaults (layer 0, mask 0xFFFFFFFF). No-op if already present.
Layer.remove(e)Remove the LayerComponent. Entity reverts to layer 0 / all-mask on next frame.
Layer.getLayer(e)intLayer index (0–31). Returns 0 when absent.
Layer.setLayer(e, index)Assign entity to a layer 0–31. Rebuilds physics filter data next frame.
Layer.getMask(e)intBitmask of layers this entity interacts with. Default 0xFFFFFFFF.
Layer.setMask(e, mask)Set the interaction mask as an integer bitmask.
Layer.setLayerEnabled(e, target, bool)Enable or disable interaction with a single layer index.
Layer.isLayerEnabled(e, target)boolIs interaction with the given layer index active?

Layer bitmask quick reference

Bit N = (1 << N). Layer 0 = 1 · Layer 1 = 2 · Layer 2 = 4 · Layer 3 = 8, etc.

Setup: player vs enemy layers

Lua
local LAYER_DEFAULT = 0
local LAYER_PLAYER  = 1
local LAYER_ENEMY   = 2
local LAYER_TERRAIN = 3

function OnStart()
    -- Put this entity on the player layer
    Layer.setLayer(this_entity, LAYER_PLAYER)

    -- Collides with terrain and enemies, but NOT other players
    local mask = (1 << LAYER_TERRAIN) | (1 << LAYER_ENEMY)
    Layer.setMask(this_entity, mask)
end

Toggle layer at runtime

Lua
-- Stop colliding with bullets while shield is active
function ActivateShield()
    Layer.setLayerEnabled(this_entity, LAYER_BULLETS, false)
end

function DeactivateShield()
    Layer.setLayerEnabled(this_entity, LAYER_BULLETS, true)
end
Home/Physics

Physics Utility

Scene-query utilities built on top of PhysX. Exposes raycast queries with optional layer filtering.

Physics.raycast

FunctionReturnsDescription
Physics.raycast(ox,oy,oz, dx,dy,dz, maxDist [, layerMask]) hit table Cast a ray from origin in direction, up to maxDist metres. Returns a hit table.

Hit result fields

FieldTypeDescription
hitboolTrue when the ray struck an actor
entityintEntity id of the struck actor, or 0 on miss
distancefloatDistance along the ray to the hit point
x, y, zfloatWorld-space hit position
nx, ny, nzfloatWorld-space surface normal at the hit point
layerMask is optional (default 0xFFFFFFFF — hits any layer). Pass a bitmask to restrict hits to specific layers, e.g. (1 << 2) for layer 2 only.

Basic — cast downward

Lua
function OnUpdate(dt)
    local pos = Transform.getWorldPosition(this_entity)
    local h = Physics.raycast(pos.x, pos.y, pos.z, 0, -1, 0, 10)
    if h.hit then
        print("Ground at: " .. h.distance .. " m  entity=" .. h.entity)
    end
end

Shoot — only hit enemies (layer 2)

Lua
local LAYER_ENEMY = 2

function Shoot()
    local pos = Transform.getWorldPosition(this_entity)
    local h = Physics.raycast(pos.x, pos.y, pos.z,
                               0, 0, 1, 50, 1 << LAYER_ENEMY)
    if h.hit then
        Script.call(h.entity, "TakeDamage", 25)
    end
end

Slope detection via hit normal

Lua
function OnUpdate(dt)
    local pos = Transform.getWorldPosition(this_entity)
    local h = Physics.raycast(pos.x, pos.y, pos.z, 0, -1, 0, 5)
    if h.hit then
        local slope = math.acos(h.ny) * (180 / math.pi)
        print("Floor slope: " .. slope .. " deg")
    end
end

Home/Debug

Debug Utility

Draw temporary lines, rays and collider wireframes directly in the editor viewport.
All shapes are rendered via the VulkanGizmo LINE_LIST pipeline — zero cost outside the editor.
Lines with duration = 0 are cleared after the frame they are drawn; any positive value keeps them alive.

Collider wireframes (Box / Sphere / Capsule) are drawn automatically for every entity that has a ColliderComponent. No Lua code is needed for those — just add a collider and the gizmo appears in the editor.

Functions

FunctionReturnsDescription
Debug.drawLine(x1,y1,z1, x2,y2,z2 [,r,g,b,a [,dur]]) Draw a segment from point A to point B. Color defaults to green (0,1,0,1). dur = seconds to keep visible; 0 = one frame only.
Debug.drawRay(ox,oy,oz, dx,dy,dz [,r,g,b,a [,dur]]) Draw a ray starting at origin with direction vector (dx,dy,dz). The magnitude of the direction determines the drawn length. Same color/duration options.
Debug.log(msg) Prints [Lua] msg to stdout. Convenient shorthand for quick tracing.

Color & duration reference

ParameterTypeDefaultNotes
r, g, b, afloat0, 1, 0, 1RGBA in 0–1 range. Alpha controls transparency.
durationfloat0.0Seconds the line stays alive. 0 = removed after it is drawn once.

Visualise a raycast hit

Lua
function OnUpdate(dt)
    local pos = Transform.getWorldPosition(this_entity)
    local h = Physics.raycast(pos.x, pos.y, pos.z, 0, 0, 1, 10)

    if h.hit then
        -- Green: origin → hit point
        Debug.drawLine(pos.x, pos.y, pos.z,
                         h.x, h.y, h.z,
                         0, 1, 0, 1)
        -- Red: surface normal (0.5 m)
        Debug.drawRay(h.x, h.y, h.z,
                        h.nx * 0.5, h.ny * 0.5, h.nz * 0.5,
                        1, 0, 0, 1)
    else
        -- Yellow: ray missed, draw full length
        Debug.drawRay(pos.x, pos.y, pos.z, 0, 0, 10,
                        1, 1, 0, 1)
    end
end

Persistent origin axes (3 seconds)

Lua
function OnStart()
    local p = Transform.getWorldPosition(this_entity)
    -- X axis — red
    Debug.drawRay(p.x, p.y, p.z,  1, 0, 0,  1, 0, 0, 1,  3.0)
    -- Y axis — green
    Debug.drawRay(p.x, p.y, p.z,  0, 1, 0,  0, 1, 0, 1,  3.0)
    -- Z axis — blue
    Debug.drawRay(p.x, p.y, p.z,  0, 0, 1,  0, 0, 1, 1,  3.0)
end

Home/Camera

Camera Component

Marks any entity as a game camera. Position and orientation are driven automatically from the entity's TransformComponent — move or rotate the entity and the view follows.
Only the entity with isActive = true overrides the renderer's main camera each frame. Switching between cameras is instant: just call Camera.setActive(e) on the desired camera entity.

A CameraComponent is attached by Camera.add(e) or automatically by Camera.setActive(e). In the Player runtime the component is read every frame — no restart required.

Functions

FunctionReturnsDescription
Camera.hasCamera(e)boolEntity has a CameraComponent
Camera.add(e)Attach a CameraComponent (fov=45, near=0.1, far=500, inactive). No-op if already present.
Camera.remove(e)Detach the CameraComponent.
Camera.setActive(e)Make e the active camera and deactivate all others. Auto-adds the component if missing.
Camera.getActive()entity idEntity id of the active camera, or 0 if none.
Camera.isActive(e)boolIs this entity the active camera?
Camera.getFOV(e)float°Vertical field-of-view in degrees.
Camera.setFOV(e, deg)Set vertical FOV (e.g., 45–120 degrees).
Camera.getNear(e)floatNear clipping plane in metres.
Camera.setNear(e, v)Set near clip (minimum ~0.01 m).
Camera.getFar(e)floatFar clipping plane in metres.
Camera.setFar(e, v)Set far clip.
Camera position and look direction come entirely from the entity's TransformComponent. To move or rotate the camera, use Transform.setPosition / Transform.setRotation on the camera entity — not any Camera function.

Setup — attach camera on start

Lua
function OnStart()
    -- This entity IS the camera: add the component and activate it
    Camera.add(this_entity)
    Camera.setFOV(this_entity, 75)
    Camera.setNear(this_entity, 0.1)
    Camera.setFar(this_entity,  1000)
    Camera.setActive(this_entity)
end

Third-person follow camera

Lua — camera_follow.lua
target = 0             -- set this to the player entity id from another script
local offset = { x=0, y=3, z=-8 }

function OnStart()
    Camera.add(this_entity)
    Camera.setFOV(this_entity, 70)
    Camera.setActive(this_entity)
end

function OnUpdate(dt)
    if target == 0 or not Entity.isValid(target) then return end
    local tp = Transform.getWorldPosition(target)
    Transform.setPosition(this_entity,
        tp.x + offset.x,
        tp.y + offset.y,
        tp.z + offset.z)
    -- Point the camera at the target
    local dx = tp.x - (tp.x + offset.x)
    local dz = tp.z - (tp.z + offset.z)
    local yaw = math.deg(math.atan(dx, -dz))
    Transform.setRotation(this_entity, 15, yaw, 0)
end

Switch cameras on trigger

Lua
cinematicCam = 0   -- entity id of the cinematic camera (set via Script.set)

function OnTriggerEnter(other)
    if cinematicCam ~= 0 then
        Camera.setActive(cinematicCam)
    end
end

function OnTriggerExit(other)
    Camera.setActive(this_entity)   -- return to player camera
end

FOV zoom effect

Lua
local normalFOV  = 70
local zoomFOV    = 20
local isZooming  = false
local currentFOV = normalFOV

function OnStart()
    Camera.add(this_entity)
    Camera.setActive(this_entity)
end

function OnUpdate(dt)
    local targetFOV = isZooming and zoomFOV or normalFOV
    currentFOV = currentFOV + (targetFOV - currentFOV) * math.min(dt * 8, 1)
    Camera.setFOV(this_entity, currentFOV)
end

function StartZoom()  isZooming = true  end
function StopZoom()   isZooming = false end

Home/CharacterController

CharacterController Component

A capsule-based kinematic controller built on PhysX sweep collision. It slides along surfaces, climbs steps and respects slope limits without requiring a Rigidbody.
Call CharacterController.move(e, dx, dy, dz) every frame with your computed displacement (gravity + input multiplied by dt). The engine performs a sweep, resolves contacts and writes the final position back to the entity's TransformComponent.

A CharacterControllerComponent is mutually exclusive with ColliderComponent + RigidbodyComponent on the same entity. Add it in the Inspector or via CharacterController.add(e).

Functions

FunctionReturnsDescription
CharacterController.has(e)boolEntity has a CharacterControllerComponent.
CharacterController.add(e)Attach with default values (radius 0.3, height 1.8, step 0.3, slope 45°).
CharacterController.remove(e)Detach and release the PhysX controller.
CharacterController.move(e, dx, dy, dz)Sweep-move by a world-space displacement. Writes back position & grounded state.
CharacterController.isGrounded(e)boolTrue when the bottom of the capsule is touching a surface.
CharacterController.getVelocity(e)x,y,zLast-frame displacement (three floats).
CharacterController.getRadius(e)floatCapsule radius in metres.
CharacterController.setRadius(e, v)Resize capsule radius (rebuilds PhysX controller).
CharacterController.getHeight(e)floatTotal capsule height in metres.
CharacterController.setHeight(e, v)Resize capsule height (rebuilds PhysX controller).
CharacterController.getSlopeLimit(e)float°Maximum walkable slope angle in degrees.
CharacterController.setSlopeLimit(e, deg)Set slope limit (0–89).
CharacterController.getStepOffset(e)floatMaximum step height the controller auto-climbs.
CharacterController.setStepOffset(e, v)Set step offset in metres.
Gravity is your responsibility. The engine does not apply gravity automatically. Accumulate a downward velocity each frame (vy = vy + (-9.81) * dt) and pass it as the dy argument to move.

Basic first-person movement

Lua
local vy      = 0      -- vertical velocity (gravity accumulator)
local speed   = 5      -- m/s horizontal speed
local jumpSpd = 6      -- m/s jump impulse
GRAVITY = -9.81

function OnStart()
    CharacterController.add(this_entity)
end

function OnUpdate(dt)
    local grounded = CharacterController.isGrounded(this_entity)
    if grounded then vy = 0 end
    vy = vy + GRAVITY * dt

    local dx, dz = 0, 0
    if Input.isKeyHeld("W") then dz = dz + speed * dt end
    if Input.isKeyHeld("S") then dz = dz - speed * dt end
    if Input.isKeyHeld("A") then dx = dx - speed * dt end
    if Input.isKeyHeld("D") then dx = dx + speed * dt end

    if grounded and Input.isKeyPressed("Space") then
        vy = jumpSpd
    end

    CharacterController.move(this_entity, dx, vy * dt, dz)
end

Coyote time & jump buffer

Lua
local vy          = 0
local coyoteTimer = 0
local COYOTE_TIME = 0.12  -- seconds of grace after walking off a ledge

function OnUpdate(dt)
    if CharacterController.isGrounded(this_entity) then
        vy = 0
        coyoteTimer = COYOTE_TIME
    else
        coyoteTimer = coyoteTimer - dt
    end
    vy = vy + (-9.81) * dt

    local canJump = coyoteTimer > 0
    if canJump and Input.isKeyPressed("Space") then
        vy = 7
        coyoteTimer = 0
    end

    CharacterController.move(this_entity, 0, vy * dt, 0)
end

Example — WASD movement with jump

Lua
local speed  = 5
local jumpVel = 8
local gravity = -20
local vy      = 0

function OnUpdate(dt)
    local h = Input.getAxis("Horizontal")   -- A/D
    local v = Input.getAxis("Vertical")     -- W/S

    if CharacterController.isGrounded(this_entity) then
        vy = 0
        if Input.getKeyDown(Input.KEY_SPACE) then
            vy = jumpVel
        end
    else
        vy = vy + gravity * dt
    end

    CharacterController.move(this_entity, h * speed * dt, vy * dt, v * speed * dt)
end

function OnUpdate(dt)

        log("Shoot! mouse moved: " .. dx .. ", " .. dy)
    end
end

Example — footstep sounds

Lua
local stepTimer = 0
local STEP_INTERVAL = 0.45

function OnUpdate(dt)
    local h = Input.getAxis("Horizontal")
    local v = Input.getAxis("Vertical")
    local moving = math.abs(h) + math.abs(v) > 0.1

    if moving and CharacterController.isGrounded(this_entity) then
        stepTimer = stepTimer + dt
        if stepTimer >= STEP_INTERVAL then
            stepTimer = 0
            AudioSource.play(this_entity)
        end
    else
        stepTimer = 0
    end
end

function OnStart()

    AudioSource.setMaxDistance(this_entity, 20)
    AudioSource.setLoop(this_entity, true)
    AudioSource.play(this_entity)
end
